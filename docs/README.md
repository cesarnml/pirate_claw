# Pirate Claw Docs

These docs describe the phase 01 MVP as a learning-oriented, review-friendly build.

Read in this order:

1. `start-here.md`
2. `phase01-mvp-design.md`
3. `phase01-implementation-phases.md`
4. `tdd-workflow.md`

Core rule for this repo:

- Do not implement all of phase 01 in one pass.
- Ship one small red-green-refactor slice at a time.
- Each ticket should be reviewable in roughly 1-3 hours of human-equivalent work.
