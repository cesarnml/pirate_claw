Pirate Claw

The DSM package installed the Pirate Claw launcher and placed the stack artifact under /volume1/pirate-claw when DSM allowed it.

Open Pirate Claw

DSM 7.1 Docker GUI first run

  1. In File Station, confirm /volume1/pirate-claw exists. If it does not, create the shared folder path and subfolders shown in the install guide.
  2. Confirm compose.synology.yml is in /volume1/pirate-claw. If DSM could not copy it during package install, upload the release-bundle copy there with File Station.
  3. In Docker, import or create the three services from the release bundle instructions: pirate-claw-web, pirate-claw-daemon, and transmission.
  4. Mount the Pirate Claw folders through Docker's volume UI. Do not type daemon tokens, Transmission credentials, JSON config, or shell commands.
  5. Start the containers in Docker, then use the button above to open Pirate Claw on port 8888.
